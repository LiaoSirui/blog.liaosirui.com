现在有两个 team

```yaml
Cohort(team-ab)
├── team-a-cq（保底 4 核，最多借到 6 核）
│     └── team-a-queue
└── team-b-cq（保底 2 核，最多借到 5 核）
      └── team-b-queue
```

给节点打 label 和 taint

```bash
# 打 label，标记节点类型
kubectl label node kube-wk-node node-type=cpu

# 加 taint，防止没有容忍的 Pod 调度上来
kubectl taint node kube-wk-node dedicated=team:NoSchedule
```

创建 ResourceFlavor（tolerations 自动模式）

```yaml
---
apiVersion: kueue.x-k8s.io/v1beta2
kind: ResourceFlavor
metadata:
  name: cpu-flavor
spec:
  nodeLabels:
    node-type: cpu # 匹配节点
  tolerations: # Kueue 自动注入 Pod，容忍节点污点
    - key: dedicated
      operator: Exists
      effect: NoSchedule

```

创建 Cohort + 两个 ClusterQueue + 两个 LocalQueue

```yaml
---
# Cohort：让两个队列共享配额
apiVersion: kueue.x-k8s.io/v1beta2
kind: Cohort
metadata:
  name: team-ab

---
# team-a：保底 4 核，最多借到 6 核
apiVersion: kueue.x-k8s.io/v1beta2
kind: ClusterQueue
metadata:
  name: team-a-cq
spec:
  cohortName: team-ab
  namespaceSelector: {}
  resourceGroups:
    - coveredResources: ["cpu", "pods"]
      flavors:
        - name: cpu-flavor
          resources:
            - name: cpu
              nominalQuota: 4
              borrowingLimit: 2
            - name: pods
              nominalQuota: 3

---
# team-b：保底 2 核，最多借到 5 核
apiVersion: kueue.x-k8s.io/v1beta2
kind: ClusterQueue
metadata:
  name: team-b-cq
spec:
  cohortName: team-ab
  namespaceSelector: {}
  resourceGroups:
    - coveredResources: ["cpu", "pods"]
      flavors:
        - name: cpu-flavor
          resources:
            - name: cpu
              nominalQuota: 2
              borrowingLimit: 3
            - name: pods
              nominalQuota: 3

---
apiVersion: kueue.x-k8s.io/v1beta2
kind: LocalQueue
metadata:
  name: team-a-queue
spec:
  clusterQueue: team-a-cq

---
apiVersion: kueue.x-k8s.io/v1beta2
kind: LocalQueue
metadata:
  name: team-b-queue
spec:
  clusterQueue: team-b-cq

```

提交 Job A（team-a，3 核，保底内）

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: job-a
  labels:
    kueue.x-k8s.io/queue-name: team-a-queue
spec:
  template:
    spec:
      containers:
        - name: sleep
          image: docker.io/library/busybox:1.38.0
          command: ["sleep", "30"]
          resources:
            requests:
              cpu: "3"
      restartPolicy: Never
```

30 秒后 Job A 跑完，验证 Kueue 自动注入了 toleration 和 nodeSelector

提交 Job B（team-b，3 核，需要借用）

team-b 保底只有 2 核，要 3 核需要从 team-a 借 1 核空闲配额（此时 Job A 还在跑，用了 3 核，team-a 空闲 1 核）。 此时提交 JobB：

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: job-b
  labels:
    kueue.x-k8s.io/queue-name: team-b-queue
spec:
  template:
    spec:
      containers:
        - name: sleep
          image: docker.io/library/busybox:1.38.0
          command: ["sleep", "30"]
          resources:
            requests:
              cpu: "3"
      restartPolicy: Never

```

借用过程：team-a 保底 4 核用了 3 核，空闲 1 核 → team-b 保底 2 核不够（要 3 核）→ 从 team-a 借了 1 核 → 准入成功。

30 秒后 Job B 跑完，team-b 的 3 核配额归还。等 Job A 也跑完后，整个 Cohort 就空了：team-a 保底 4 核全空闲，team-b 保底 2 核全空闲，总共 6 核可借。

提交 Job C（team-b，5 核，测试 borrowingLimit 上限）

Job A、Job B 都已跑完，配额全部归还。现在 team-b 最多能用保底 2 + 借 3 = 5 核（team-a 空闲 4 核，足够借）。

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: job-c
  labels:
    kueue.x-k8s.io/queue-name: team-b-queue
spec:
  template:
    spec:
      containers:
        - name: sleep
          image: docker.io/library/busybox:1.38.0
          command: ["sleep", "120"]
          resources:
            requests:
              cpu: "5"
      restartPolicy: Never

```

提交 Job D（team-b 再多 1 核，超过 borrowingLimit）

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: job-d
  labels:
    kueue.x-k8s.io/queue-name: team-b-queue
spec:
  template:
    spec:
      containers:
        - name: sleep
          image: docker.io/library/busybox:1.38.0
          command: ["sleep", "120"]
          resources:
            requests:
              cpu: "1"
      restartPolicy: Never

```

team-b 最多 5 核，Job C 已用 5 核，没有余量，无法调度，Job 被调整为 Suspended
