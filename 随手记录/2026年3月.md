<https://www.rockylinux.cn/notes/ldap-account-management-freeipa-add-remove-domain-chapter.html>

配置参考

```ini
[sssd]
services = nss, pam
domains = ldap.alpha-quant.tech

[domain/ldap.alpha-quant.tech]
id_provider = ldap
auth_provider = ldap
ldap_uri = ldap://ldap.alpha-quant.tech
ldap_search_base = DC=alpha-quant,DC=tech 
ldap_id_use_start_tls = False
ldap_tls_reqcert = never
cache_credentials = True
fallback_homedir = /home/%u
use_fully_qualified_names = False

# 限制登录
access_provider = simple
simple_allow_users = srliao
```

配置允许的用户在：`/etc/sssd/sssd.conf`，配置项：`simple_allow_users = srliao`

配置后先检查：`sssctl config-check`，确定没有问题再重启 `systemctl restart sssd`

- `/etc/security/group.conf`

添加以下行，赋予普通用户组 sudo 权限：

```
*; *; *; Al0000-2400; wheel
```

在  `/etc/pam.d/sshd`中添加：

```bash
auth required pam_group.so minimum_uid=1000
```

或者更为激进的

```bash
ALL ALL=(ALL) NOPASSWD: ALL
ALL ALL=(ALL) ALL

```
